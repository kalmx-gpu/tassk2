Проверка на работоспособность index.php.

1. Получить главную страницу методом GET в протоколе HTTP 1.0

Отправлен HTTP-запрос GET / HTTP/1.0 к главной странице сервера. Запрос выполнен через telnet, получен ответ с заголовками и содержимым главной страницы.

2. Получить внутреннюю страницу методом GET в протоколе HTTP 1.1

Отправлен запрос GET /index.php HTTP/1.1 с обязательным заголовком Host: u82290.kubsu-dev.ru. Сервер вернул код 200 OK и содержимое страницы.

3. Определить размер файла file.tar.gz, не скачивая его

Использован метод HEAD /tassk2/file.tar.gz HTTP/1.1. В ответе получен заголовок Content-Length: 11335, что соответствует размеру файла в байтах. Сам файл не передавался.

4. Определить медиатип ресурса /image.png

Отправлен HEAD /tassk2/image.png HTTP/1.0. В ответе найден заголовок Content-Type: image/png, указывающий MIME-тип файла.

5. Отправить комментарий на сервер по адресу /index.php

Выполнен POST-запрос к /index.php с телом comment=Test+123. Заголовки: Content-Type: application/x-www-form-urlencoded и Content-Length: 18. Сервер ответил 200 OK, обработав запрос.

6. Получить первые 100 байт файла /file.tar.gz

Отправлен GET-запрос с заголовком Range: bytes=0-99. Сервер ответил статусом 206 Partial Content, вернув 100 байт данных и заголовок Content-Range: bytes 0-99/11335.

7. Определить кодировку ресурса /index.php

Использован HEAD-запрос к /index.php. В заголовке Content-Type указано charset=UTF-8, что означает кодировку UTF-8 для передаваемого документа.
